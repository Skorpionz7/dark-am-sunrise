# Security Policy

## Supported Versions
We provide security updates for the latest major release.

## Reporting a Vulnerability
If you discover a security issue, please open a **private security advisory** using GitHub’s “Report a vulnerability” feature.  
Do not create a public issue or pull request. You can see it for instance, under "Advisories".

## Disclosure Process
After receiving a private report:
- We investigate and confirm the issue.
- A fix is developed and released.
- The reporter is updated through the private advisory thread.
- A public security advisory is published once users have had time to update.

## Security Best Practices
- Keep your environment and dependencies updated.
- Avoid sharing sensitive information in public
